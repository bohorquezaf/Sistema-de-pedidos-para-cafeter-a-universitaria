# ¿Qué es Markdown y por qué se utiliza en proyectos de software?

Markdown es un lenguaje de marcado ligero que permite crear documentos con formato utilizando una sintaxis simple y fácil de leer.  
Fue creado por John Gruber con el objetivo de que el texto sea legible incluso sin procesarlo, a diferencia de HTML u otros lenguajes más complejos.

En proyectos de software se utiliza porque:

- Permite documentar de forma clara y ordenada.
- Es rápido de escribir y leer.
- Evita distracciones al no requerir herramientas complejas.
- Se integra perfectamente con GitHub, GitLab, Bitbucket y otras plataformas de control de versiones.
- Es ideal para documentación técnica, README, wikis, issues, manuales y notas de desarrollo.
- Puede convertirse a PDF, HTML o DOCX fácilmente.

# Ventajas de utilizar Markdown en combinación con GitHub

- Documentación viva: Los archivos .md se renderizan automáticamente en GitHub.
- Control de versiones: Puedes ver quién cambió qué y cuándo.
- Colaboración: Markdown permite que varios desarrolladores editen documentación fácilmente.
- Integración con GitHub Pages: Puedes convertir Markdown en sitios web estáticos.
- Compatibilidad: Issues, Pull Requests y README usan Markdown por defecto.
- Estándar de la industria: Es el formato favorito para documentación técnica moderna.
- Ligero y portable: Funciona en cualquier editor de texto.
